# 改进方案测试任务

> 本文档基于《改进方案.md》中的问题与改进点，列出相应的测试任务，供回归与验收使用。

---

## 2. 学生端：课程修读与成绩查询

### 2.1.1.1 学生选修/退课
- [ √ ] 在课程浏览页点击退课按钮，验证可以正常触发退课操作，不再出现“无响应”情况。
- [ √ ] 在弹出的退课确认提示窗口中选择“取消”，验证不会重复添加选课卡片。
- [ √ ] 在确认退课后，验证该课程从“我的课程”列表中消失，且后端选课记录状态正确更新。

### 2.1.3 课程学习任务
- [ √ ] 在教师端为某门课程发布作业/考试，登录学生端后，验证该课程下能正确看到对应任务列表。
- [ √ ] 在学生端对任务以文本方式提交作业，验证提交成功提示正常显示，且教师端能看到该提交记录。
- [ √ ] 在学生端以文件上传方式提交作业，验证提交成功/失败提示信息展示正确（包括失败时错误原因）。
- [ ] 在教师端对学生提交的作业/考试进行批阅打分，验证分数能保存并在学生端成绩详情中正确体现。
- [ ] 管理员在教学管理端新建一门课程后，刷新学生端首页，验证首页课程列表中能看到新创建的课程。

### 2.2.2 课程成绩详情布局
- [ ] 打开课程成绩详情页，验证各“成绩项信息”与对应“上分栏目”在界面上对齐显示，无错位和重叠。
- [ ] 在不同分辨率（PC、较窄窗口）下重复验证布局，确保响应式布局下仍对齐。

---

## 3. 教师端：课程与成绩管理

### 3.1.1 授课列表与创建课程
- [ ] 在教师端通过“创建课程”入口填写课程代码、课程名称、学分、学期、先修课、课程描述等信息并保存，验证创建成功。
- [ ] 创建课程后点击“查询授课列表”，验证新课程出现在授课列表中，字段包括授课任务ID、学期、课程ID、课程代码、课程名称。

### 3.1.2 上传课程资料
- [ ] 打开上传课程资料页面，验证课程选择使用“课程名称”下拉框，而非直接输入课程ID。
- [ ] 为某课程上传文档或视频资料，验证上传成功提示信息正常显示。
- [ ] 上传成功后，验证在资料区域新增卡片，卡片上包含：资料名称、上传者姓名/账号、上传时间、资料大小、下载链接。
- [ ] 对已上传的资料卡片执行“编辑”操作，修改标题等字段并保存，验证页面和后端数据均更新正确。
- [ ] 对已上传的资料卡片执行“撤回/删除”操作，验证卡片从界面移除且下载链接失效。

### 3.1.3 更新课程配置
- [ ] 在课程配置页面，验证课程选择来源于“教师授课列表”的下拉框，而非手动输入课程ID。
- [ ] 选择某课程后，界面应自动展示当前配置（简介、是否开启评论区、是否开启笔记区），验证与后端数据一致。
- [ ] 修改简介和评论区开关等配置并保存，验证前端显示与后端数据一致，刷新页面后仍保持修改结果。

### 3.2.1 布置作业或考试
- [ ] 在布置作业/考试页面，验证课程选择来源于“教师授课列表”下拉框。
- [ ] 填写作业/考试标题、描述、截止日期及附件后提交，验证布置成功提示信息正确显示。
- [ ] 布置成功后，验证对应区域中新建一张作业/考试卡片，展示：课程名称、标题、截止日期、创建时间以及附件下载链接（如有）。

### 3.2.2 查看作业/考试提交情况
- [ ] 在教师端打开某作业的提交情况页面，验证列表中展示学生ID、学生姓名、提交时间、作业状态（已提交/未提交）。
- [ ] 通过学生ID筛选或定位某学生，验证可以查看其该作业的完成情况。
- [ ] 在提交列表中为某学生输入或修改分数并保存，验证分数持久化成功，并在成绩管理中相应成绩项中能看到该分数（与 3.3.2 录入单个成绩功能联动）。
- [ ] 在作业信息卡片上显示“已提交人数/全部人数”，验证两个数字与实际提交情况一致。

### 3.3.1 设置成绩组成项
- [ ] 在成绩组成设置页面，验证课程选择来源于“教师授课列表”下拉框。
- [ ] 进入某课程后，界面展示当前成绩构成列表（成绩项名称、权重、描述），验证与后端数据一致。
- [ ] 在当前权重和为小于等于 1 的情况下新增成绩项，验证可以正常保存且权重总和正确更新。
- [ ] 当尝试新增一个成绩项导致“旧成绩项权重和 + 新成绩项权重 > 1”时，验证前端弹出错误信息并阻止提交。
- [ ] 修改已有成绩项的权重，使总和重新不超过 1 后，再次新增成绩项，验证可以正常通过并保存。

## 4. 教学管理端

### 4.1.1 班级信息管理
- [ ] 在教学管理端创建班级，验证创建功能依旧正常（回归测试）。
- [ ] 在班级列表中对某条记录点击“编辑”，修改字段并保存，验证数据更新成功且前端列表同步刷新。
- [ ] 在班级列表中对某条记录点击“删除”，验证删除确认流程与最终删除结果正确（可使用软删除或物理删除策略）。

### 4.1.2 学生信息管理
- [ ] 使用搜索框按学号、姓名或班级进行搜索，验证可正确过滤出匹配的学生记录。
- [ ] 对搜索结果或列表中的学生执行“编辑”操作，保存后验证信息更新正确。
- [ ] 对某学生执行“删除”操作，验证删除流程与结果正确。
- [ ] 切换某学生的“激活/锁定”状态，验证状态切换成功并在界面中实时反映（例如状态标签或开关样式变化），并在后端数据中正确记录。

### 4.1.3 教师信息管理
- [ ] 使用搜索框按姓名、工号等条件搜索教师，验证搜索结果准确。
- [ ] 在教师列表中验证“工号”字段正确显示。
- [ ] 对教师记录执行“编辑”和“删除”操作，确认交互生效且数据正确更新。

### 4.1.4 课程信息管理
- [ ] 在课程列表模块中验证搜索功能（按课程代码、名称、院系等）可正常使用。
- [ ] 对课程记录进行“编辑”操作，保存后确认列表与详情中均已更新。
- [ ] 对课程记录执行“删除”操作，验证按钮交互生效并正确更新数据（与 4.1.2、4.1.3 一致的交互体验）。

### 4.1.5 教室信息管理
- [ ] 在教室管理模块中验证搜索、编辑、删除操作与班级/学生/教师模块一致，均能正常交互并更新后端数据。

### 4.3.1 获取待审核的成绩
- [ ] 打开“待审核成绩”页面，验证查询按钮、分页、搜索条件（课程名称、学期等）均可正常使用。
- [ ] 验证每条记录可点击进入详情或进行审核操作，而非按钮“无响应”。

### 4.3.2 批量发布成绩
- [ ] 在“批量发布成绩”页面，使用搜索和筛选功能定位目标课程，验证可正常筛选。
- [ ] 勾选多门课程后执行“批量发布”，确认按钮有响应，操作成功后返回“发布成功课程数量”等摘要信息，并可在相关模块中看到成绩状态变为“已发布”。

## 5. 系统管理端

### 5.1.1 查询操作日志样式
- [ ] 打开系统管理端的操作日志页面，验证功能使用正常（查询、分页、过滤等）。
- [ ] 检查左侧侧边栏中“系统管理”和“其他”菜单项的文字颜色是否比默认更显眼，符合设计预期。
- [ ] 在深色模式/浅色模式或不同主题下，验证该颜色仍保持良好的可读性和对比度。
